# 📊 Лабораторна робота: Алгоритми обробки числових послідовностей на C++

## 🛠 Що робить програма?

Програма реалізує повний цикл обробки даних: від генерації до експорту у структуровані формати.

| Етап | Опис процесу |
| :--- | :--- |
| **1. Введення** | Запит кількості елементів (`size`) та кроку групування (`n`). |
| **2. Генерація** | Створення послідовності випадкових чисел у діапазоні `[1; 99]`. |
| **3. Групування** | Розрахунок сум елементів порціями по `n` одиниць та запис у `e.txt`. |
| **4. Сортування** | Впорядкування масиву за зростанням (`ASC`) та спаданням (`DESC`). |
| **5. Очищення** | Видалення дублікатів та формування масиву унікальних значень. |
| **6. Експорт** | Збереження фінальних показників (S2, S3, різниця) у `data.json`. |

---

## 📂 Структура файлів проєкту

*   📄 **`project1.cpp`** — Головний файл (точка входу), логіка взаємодії з користувачем.
*   📜 **`functions.h`** — Бібліотека функцій обробки (всі функції позначені як `inline` для оптимізації).
*   📁 **`MyProject.Tests`** — Набір успішних тестів для перевірки логіки. ✅
*   📁 **`MyProject.NegativeTests`** — Спеціальний модуль із навмисно помилковим тестом (демонстрація `Fail`). ❌

---

## 🧪 Модульне тестування (Unit Testing)

Проєкт використовує фреймворк **Microsoft Native Unit Test**. Тести розділені на два незалежні проєкти для кращої візуалізації в **Test Explorer**:

1.  **Positive Tests:** Перевірка стандартних сценаріїв (сума $1+2+3=6$).
2.  **Negative Tests:** Перевірка реакції системи на невідповідність очікуваним даним.

## 📊 Формат вихідних даних (`data.json`)

```json
{
  "S2": "Загальна сума елементів",
  "S3": "Сума унікальних елементів",
  "S3 - S2": "Різниця між сумами",
  "duplicatesCount": "Кількість знайдених повторів",
  "V1": [ "Початковий масив" ],
  "V2": [ "Масив без дублікатів" ]
}
```
